[스크립트]
seconds
	인자로 초를 넣어주면 게임 속도에 맞춰 스텝 단위로 반환해주는 함수입니다.

ease_in_expo
	보간 함수입니다. 처음에 빠르게 증가하다가 나중에는 천천히 증가하는 지수 함수입니다. 0 ~ 1의 값을 넣으면 
	0 ~ 1의 값을 반환합니다.

menu_item_clear
	어떤 깊이의 내용물을 모두 초기화하고 기본적으로 선택할 항목을 정합니다. 인자를 주지 않으면 기본 선택 
	항목은 0(첫번째) 입니다.

menu_item_add
	반드시 menu_item_clear 실행 후에 쓰셔야 합니다. 그렇지 않으면, 만약 이전에 해당 깊이에 다른 항목을 추가했던 
	적이 있다면 불필요한 항목들이 점점 늘어나게 됩니다. 스크립트가 각 항목에 연결되어 동작에 문제는 없지만 
	정상 상태라고 볼 수는 없습니다. 그리고 이전에 사용한 적 없던 깊이에 항목을 추가하면 오류가 발생합니다.

[글꼴]
fontMainMenu, fontMainMenuBold
	두 글꼴은 반드시 같은 글자 범위를 갖고있어야합니다. 메인 메뉴를 그리는데 사용합니다.

[객체]
oInit
	초기화 용 객체입니다. 예시용 여러 전역 변수들을 선언합니다. 프로필, 도전과제, 설정 변수들을 선언합니다.

oMainMenu
	깊이 기반 메뉴를 그려주는 객체입니다. 상하좌우 방향키로 메뉴 항목을 이동하고, 엔터, 스페이스 바, A 키로
	항목을 선택합니다. 백스페이스 키로 이전 깊이로 갈 수 있습니다.

	보통 볼 수 있는 선형적 메뉴와는 완전 다른 방식으로 작동합니다. 메뉴 항목들을 구분하는 '깊이'라는 단위가 
	하나 더 있습니다. 일종의 범주(카테고리) 와 같습니다. 어떤 항목을 선택하면 바로 어떤 내용을 실행하는 경우가
	있고, 혹은 다음 깊이로 넘어가는 경우가 있습니다. 이 모든 일련의 동작은 콜백(Callback) 이라고 불리는 
	스크립트들에 의해 이루어집니다. 게임메이커의 변수가 스크립트를 담을 수 있다는 점에서 착안한 방식입니다.
	직접 하드 코딩하는 것보다 훨씬 더 효율적입니다. 파이썬, 자바스크립트, C# 등지에서 매우 자주 볼 수 있는 
	방식입니다.

	다음 깊이로 넘어가는 경우, 이때 선택했던 하나의 메뉴 항목이 그 전환되는 깊이의 이름(범주) 가 됩니다. 그리기 
	이벤트에서는 이전에 선택한 항목은 페이드 아웃하여 사라지지 않고, 그대로 새로 표시될 메뉴 항목들 왼쪽에 
	남아있게 됩니다.

	자세한 내용은 직접 코드를 확인하시길 바랍니다. 하지만 알아두셔야 할 것은 굳이 이 코드의 내용을 전부 이해할 
	필요는 없다는 것입니다. 생성 이벤트에서 시연용 메뉴 항목 여러개를 추가하는 부분이 있습니다. 여러분은 
	그저 이 부분에 여러분이 원하시는 메뉴 항목들을 추가하시면 됩니다. 그렇지만 이 예제를 사용하시면서 가장 
	문제가 되는 부분은 콜백들을 추가하고 수정하는 부분일 겁니다. 얼핏 비효율적이고 노가다라고 느껴지실지도 
	모르겠지만 이 방법 외에 다른 방식들은 훨씬 복잡하고 힘듭니다. 사용자 이벤트를 이용하는 게 그나마 나은 
	방법에 속합니다.

[룸]
roomInit
	초기화 용 룸입니다. oInit 만이 배치되있습니다.
